☕ Java Prüfungsvorbereitung

Umfassender Lernleitfaden für Fachinformatiker (Anwendungsentwicklung)

Die Beherrschung der Programmiersprache Java ist ein fundamentaler Bestandteil der Ausbildung zum Fachinformatiker für Anwendungsentwicklung. Java ist eine objektorientierte, streng typisierte und plattformunabhängige Sprache, die 1995 von Sun Microsystems entwickelt wurde und heute von Oracle gepflegt wird. Sie zeichnet sich durch ihr architektonisches Leitmotiv "Write Once, Run Anywhere" (WORA) aus.

Dieser Leitfaden ist kognitionspsychologisch optimiert. Alle Quellcode-Beispiele sind mittels Prism.js tokenisiert und syntaktisch hervorgehoben. Diese semantische Farbgebung reduziert die kognitive Belastung (Cognitive Load) drastisch und ermöglicht es dem Auge, Schlüsselwörter, Datentypen und Logikblöcke präattentiv zu erfassen.

Anatomie eines Basis-Programms Jedes ausführbare Java-Programm erfordert zwingend eine Klassendefinition sowie die Deklaration der Hauptmethode public static void main(String args). Das Schlüsselwort static indiziert hierbei, dass der Interpreter die Methode direkt aus dem Speicherkontext der Klasse aufrufen kann, ohne dass im Vorfeld ein Objekt dieser Klasse instanziiert werden muss. 
// Ein minimales Java-Programm
public class HelloWorld {
    public static void main(String args) {
        System.out.println("Hello from Java!");
    }
}

2. Die Java-Architektur: JDK, JRE & JVM

Um die versprochene Hardware-Unabhängigkeit zu gewährleisten, kompiliert Java den Quellcode nicht direkt in nativen Maschinencode, sondern nutzt eine dreischichtige virtuelle Architektur. Ein tiefgreifendes Verständnis dieser Komponenten ist essenziell.

  • JDK (Java Development Kit): Dies ist das vollständige Software Development Kit (SDK) für Entwickler. Es stellt eine Obermenge der JRE dar und enthält alle notwendigen Entwicklungswerkzeuge. Die wichtigsten Tools sind der Compiler (javac), welcher den Quellcode übersetzt, sowie Debugger und Archivierungs-Tools (jar).
  • JRE (Java Runtime Environment): Die JRE ist die reine Laufzeitumgebung, die von Endanwendern benötigt wird, um kompilierte Java-Programme auszuführen. Sie enthält die Java Virtual Machine (JVM), den Class-Loader und essenzielle Basisbibliotheken, bietet jedoch keine Werkzeuge zur Softwareentwicklung.
  • JVM (Java Virtual Machine): Die JVM ist das Herzstück der Architektur und fungiert als Abstraktionsschicht zwischen dem Programm und dem Betriebssystem. Sie interpretiert den vom Compiler erzeugten Bytecode (die .class-Dateien).
Performance-Optimierung: Der JIT-Compiler
Reine Interpreter sind historisch bedingt langsam. Die JVM löst dieses Problem durch den integrierten Just-In-Time (JIT) Compiler. Dieser überwacht die Code-Ausführung zur Laufzeit. Identifiziert er wiederkehrende Sequenzen (Hotspots), übersetzt er diesen Bytecode in hochoptimierten, nativen Maschinencode der jeweiligen Hardware und speichert ihn zwischen, was die Ausführungsgeschwindigkeit massiv steigert.

3. Datentypen, Speichermanagement & Type Casting

Java unterscheidet auf fundamentaler Ebene zwischen primitiven Datentypen und Referenz- bzw. Objekttypen. Das Typsystem ist statisch, was bedeutet, dass der Datentyp einer Variablen zur Kompilierzeit unveränderlich feststehen muss.

Primitive Datentypen

Primitive Typen weisen einen fest definierten Speicherbedarf auf. Die Kenntnis dieser Grenzen verhindert Pufferüberläufe (Overflows) in der Softwarearchitektur.

Typname Speicherbedarf Wertebereich Spezifikation / Einsatzgebiet
boolean 1 Byte true, false Repräsentiert einen Booleschen Wahrheitswert. Logisch 1 Bit.
char 2 Byte (16 Bit) Unicode-Zeichensatz Speichert ein einzelnes Zeichen. Wird in einfachen Hochkommata notiert (z. B. 'a', '%').
byte 1 Byte (8 Bit) -128 bis 127 Das achte Bit fungiert als Vorzeichen. Ideal zur Speichereinsparung in großen Arrays.
short 2 Byte (16 Bit) -32.768 bis 32.767 Mittlerer Ganzzahltyp.
int 4 Byte (32 Bit) -2^31 bis 2^31 - 1 Der Standard-Datentyp für Ganzzahlen in Java (Wertebereich ca. ±2,14 Milliarden).
long 8 Byte (64 Bit) -2^63 bis 2^63 - 1 Für extrem große Zahlen. Literale benötigen das Suffix 'L' (z. B. 10000000000L).
float 4 Byte (32 Bit) Einfache Genauigkeit Fließkommazahl nach IEEE 754. Literale benötigen das Suffix 'f' (z. B. 3.14f).
double 8 Byte (64 Bit) Doppelte Genauigkeit Standard-Fließkommazahl mit maximalem Exponentenbereich.

Referenztypen (Objekttypen)

Im Gegensatz zu primitiven Typen speichern Referenztypen nicht den Wert selbst, sondern die Speicheradresse (Referenz) des Objekts auf dem Heap. Zu den Referenztypen gehören Objekte instanziierter Klassen, Arrays und Strings.

Typumwandlung (Type Casting)

Werden unterschiedliche Datentypen in Ausdrücken gemischt, müssen Konvertierungen erfolgen:

  • Implizites Casting (Widening): Erfolgt automatisch durch den Compiler, wenn verlustfrei von einem kleineren in einen größeren Datentyp konvertiert wird (z. B. short zu int).
  • Explizites Casting (Narrowing): Muss vom Entwickler manuell forciert werden, wenn Informationsverlust droht (z. B. von double zu int, wobei die Nachkommastellen unwiderruflich abgeschnitten werden).
// Beispiel für explizites Type-Casting
double kommazahl = 10.99;
int ganzzahl = (int) kommazahl; // Expliziter Cast. Der Wert von 'ganzzahl' ist nun exakt 10.

String str = "123";
int numericValue = Integer.parseInt(str); // String zu Numeric

4. Lexikalische Struktur und Operatoren-Logik

Operatoren sind die syntaktischen Werkzeuge zur Manipulation von Werten und Variablen. Java bietet ein umfangreiches Set für arithmetische und logische Berechnungen.

  • Arithmetische Operatoren: + (Addition), - (Subtraktion), * (Multiplikation), / (Division).
  • Modulo-Operator: % berechnet den ganzzahligen Rest einer Division (z. B. 7 % 3 liefert 1).
  • Inkrement / Dekrement: ++ erhöht den Wert einer Variablen um 1, -- verringert ihn um 1.
  • Relationale (Vergleichs-)Operatoren: ==, !=, <, >, <=, >=. Beachten Sie, dass == bei Referenztypen nur prüft, ob beide Referenzen auf dasselbe Objekt im Speicher zeigen. Um inhaltliche Gleichheit von Objekten (wie Strings) zu prüfen, ist die Methode .equals() zu verwenden.
  • Logische Operatoren: && (logisches UND, wertet bedingerfüllt aus), || (logisches ODER), ! (logisches NICHT).
Tipp zur Ganzzahldivision (Integer Division):
Eine häufige Fehlerquelle besteht im Umgang mit dem Divisionsoperator (/). Sind beide Operanden ganzzahlig (z. B. int), erzwingt Java eine Ganzzahldivision. Der Rest wird dabei ersatzlos gestrichen. Die Operation 7 / 3 liefert somit 2. Soll ein exaktes Fließkommaergebnis erzielt werden, muss mindestens ein Operand ein Fließkommatyp sein: 7.0 / 3 evaluiert zu 2.333333.

5. Algorithmen und Kontrollfluss

Der Programmablauf wird durch Selektionen (Entscheidungen) und Iterationen (Schleifen) gesteuert.

Selektionen (if, else, switch)

Die switch-Anweisung bietet eine effiziente Mehrfachauswahl für die Typen byte, short, int, char (und ab Java 7 auch String). Ohne ein break am Ende eines case-Blocks wird der Code des nachfolgenden Blocks ebenfalls ausgeführt (sogenannter Fall-Through).

Schleifen und Algorithmik (Code-Beispiele)

Die nachfolgenden Beispiele aus der angewandten Mathematik demonstrieren den Kontrollfluss von for- und while-Schleifen.

// Algorithmus 1: Fibonacci-Folge mittels for-Schleife
int t1 = 0, t2 = 1, sum = 0;
for (int i = 1; i <= n; ++i) {
    System.out.print(t1 + " ");
    sum = t1 + t2;
    t1 = t2;
    t2 = sum;
}

// Algorithmus 2: Fakultät einer Zahl (Rekursiver Ansatz)
// Die Methode ruft sich selbst auf, bis die Abbruchbedingung (n == 0) greift.
public int factorial(int n) {
    if (n == 0) {
        return 1;
    } else {
        return (n * factorial(n - 1));
    }
}

// Algorithmus 3: Primzahlprüfung mittels Modulo-Operator
public boolean isPrime(int n) {
    if (n < 2) return false;
    for (int i = 2; i <= n/i; ++i) {
        if (n % i == 0) return false; // Restlose Teilbarkeit bedeutet keine Primzahl
    }
    return true;
}

6. Das Paradigma der Objektorientierten Programmierung (OOP)

Während die klassische, prozedurale Programmierung Funktionen und Daten strikt trennt (was bei großen Systemen zu unübersichtlichem "Spaghetti-Code" und unbeabsichtigten Nebeneffekten führt), modelliert die OOP reale Entitäten als Objekte. Diese Objekte kapseln sowohl ihren Zustand (Attribute/Variablen) als auch ihr Verhalten (Methoden).

  • Klassen und Instanziierung: Eine Klasse (z. B. Person) ist der abstrakte Bauplan. Das Objekt (z. B. oSchueler1) ist die zur Laufzeit im Speicher liegende Instanz, die meist mit dem new-Operator erzeugt wird.
  • Kapselung (Encapsulation): Die Integrität der Objektdaten wird geschützt, indem Attribute als private deklariert werden. Der Zugriff von außen erfolgt ausschließlich über öffentliche Schnittstellen, sogenannte Getter- und Setter-Methoden. Dies verhindert unkontrollierte Zustandsänderungen.
  • Komposition (Has-a-Beziehung): Ein Objekt kann andere Objekte enthalten, um komplexe Architekturen zu bilden. Ein Beispiel ist eine Person-Klasse, die als Attribut ein Objekt der Klasse Adresse integriert.
  • Vererbung (Is-a-Beziehung): Durch das Schlüsselwort extends kann eine Subklasse (z. B. Schueler) die Attribute und Methoden einer Superklasse (z. B. Person) übernehmen und um spezifische Merkmale erweitern. Dies fördert die massive Wiederverwendbarkeit von Code.
  • Polymorphie & Überschreiben: Subklassen können geerbte Methoden ihrer Superklasse überschreiben (Method Overriding). Ruft das Programm zur Laufzeit eine Methode über eine Basisklassen-Referenz auf, entscheidet die JVM dynamisch anhand des echten Objekttyps, welche spezifische Implementierung ausgeführt wird.

7. Arrays und das Collections Framework

Datenstrukturen sind essenziell zur Verwaltung von Element-Sammlungen.

Multidimensionale Arrays (Matrizen)

Ein Array ist ein Referenztyp fester Länge. Mehrdimensionale Arrays können als Matrizen visualisiert werden. Die Matrizenmathematik erfordert ein tiefes Verständnis von geschachtelten Schleifen.

// Matrix-Transposition (Spiegeln an der Diagonale)
for(int i = 0; i < row; i++) {
    for(int j = 0; j < column; j++) {
        System.out.print(array[i][j] + " ");
    }
    System.out.println("");
}

// Multiplikation zweier Matrizen (Skalarprodukt der Zeilen und Spalten)
for (int i = 0; i < row1; i++) {
    for (int j = 0; j < col2; j++) {
        for (int k = 0; k < row2; k++) {
            sum = sum + first[i][k] * second[k][j];
        }
        multiply[i][j] = sum;
        sum = 0;
    }
}

Java Collections Framework

Da einfache Arrays in ihrer Länge starr sind, bietet das `java.util` Paket dynamische Datenstrukturen. Eine ArrayList erlaubt das zur Laufzeit dynamische Hinzufügen und Entfernen von Elementen. Ein HashSet filtert automatisch Duplikate heraus, da es eine mathematische Menge repräsentiert.

8. Robustheit: Fehlerbehandlung und Assertions

Professionelle Applikationen müssen mit unvorhergesehenen Systemzuständen umgehen können.

  • Exceptions (Ausnahmen): Laufzeitfehler (wie Division durch Null oder fehlende Dateien) werden in Java geworfen (throw). Um Programmabstürze zu verhindern, wird riskanter Code in einem try-Block überwacht. Tritt ein Fehler auf, wird der Kontrollfluss in einen korrespondierenden catch-Block umgeleitet. Ein optionaler finally-Block wird stets ausgeführt (z. B. zum Schließen von Datenbankverbindungen).
  • Assertions (Zusicherungen): Assertions (assert bedingung : "Meldung";) dienen primär dem Entwickler, um Logikfehler und Annahmen (Invariants) zur Laufzeit zu überprüfen. Sie können beim Start der JVM mittels -ea aktiviert werden und sind in der finalen Produktionsumgebung meist deaktiviert.

9. Praxis-Analyse: Die Architektur der "SchoolApp"

Die folgenden Klassen repräsentieren ein vollständiges, objektorientiertes Architektur-Beispiel aus dem Schulwesen. Die Analyse dieser Codebasis verdeutlicht das ineinandergreifen der OOP-Prinzipien Kapselung, Komposition, Vererbung und Polymorphie.

1. Kapselung: Die Datenklasse Adresse.java

Diese Klasse fungiert als reiner Datenspeicher (POJO). Durch die Modifikatoren private wird der direkte Speicherzugriff gesperrt (Kapselung). Die Hausnummer ist als String deklariert, um alphanumerische Zusätze (wie "12b") zu ermöglichen.

public class Adresse {

    private String strStrasse;
    private String strHausNr; 
    private String strPLZ;
    private String strOrt;
    
    // Öffentliche Schnittstellen (Getter und Setter)
    public String getStrStrasse() { return strStrasse; }
    public void setStrStrasse(String strStrasse) { this.strStrasse = strStrasse; }
    
    public String getStrHausNr() { return strHausNr; }
    public void setStrHausNr(String strHausNr) { this.strHausNr = strHausNr; }
    
    public String getStrPLZ() { return strPLZ; }
    public void setStrPLZ(String strPLZ) { this.strPLZ = strPLZ; }
    
    public String getStrOrt() { return strOrt; }
    public void setStrOrt(String strOrt) { this.strOrt = strOrt; }
}

2. Basisklasse: Person.java (Komposition & Überladung)

Diese Basisklasse nutzt Komposition, indem sie ein Objekt der Klasse Adresse als Attribut einbindet. Bemerkenswert ist die Klassenvariable static int iPAnzahl. Sie fungiert als globaler Zähler über alle Instanzen hinweg und garantiert im Konstruktor eine eindeutige Personen-ID. Ferner demonstriert die Methode erfasseDaten das Prinzip des Method Overloading (Methodenüberladung).

import java.util.Scanner;

public class Person {
    // statische Klassenvariable zur Generierung eindeutiger IDs
    private static int iPAnzahl = 0;  
    
    private int iPNr; // Instanzspezifische, eindeutige ID
    private String strNachName;
    private String strVorName;
    
    // Komposition: Eine Person "hat-eine" Adresse
    Adresse adresse = new Adresse();    
    
    public Person() {
        iPNr = ++iPAnzahl; // Präinkrement: Zähler erhöhen und zuweisen
    }
        
    public void erfasseDaten() {
        Scanner sc = new Scanner(System.in);
        System.out.println("Dateneingabe");
        System.out.print("Nachname:\t");
        strNachName = sc.next();
        System.out.print("Vorname:\t");
        strVorName = sc.next();
        
        // Zuweisung an das gekapselte Adress-Objekt
        System.out.print("Strasse:\t");
        adresse.setStrStrasse(sc.next());
        System.out.print("Hausnr:\t\t");
        adresse.setStrHausNr(sc.next());
        System.out.print("PLZ:\t\t");
        adresse.setStrPLZ(sc.next());
        System.out.print("Ort:\t\t");
        adresse.setStrOrt(sc.next());
    }
    
    // Methodenüberladung: Gleicher Name, unterschiedliche Signatur
    public void erfasseDaten(String strNachName) {
        this.strNachName = strNachName; // 'this' referenziert die Instanzvariable
    }
    
    public void druckeAusweis() {
        System.out.println("Personennr.:\t" + iPNr);
        System.out.println("Nachname:\t" + strNachName);
        System.out.println("Vorname:\t" + strVorName);
        System.out.println("Adresse:\t" + adresse.getStrStrasse() + " " + adresse.getStrHausNr() + 
                           "\n\t\t" + adresse.getStrPLZ() + " " + adresse.getStrOrt());
    }
}

3. Subklasse: Schueler.java (Vererbung & Überschreibung)

Die Klasse Schueler erbt mittels extends von Person. Um Codeduplizierung zu vermeiden, ruft sie innerhalb von erfasseDaten() über das Schlüsselwort super zunächst die Logik der Elternklasse auf. Die Methode druckeAusweis() wird spezifisch für den Schüler überschrieben.

import java.util.Scanner;

// Vererbung: Schueler erbt das gesamte Verhaltensmuster von Person
public class Schueler extends Person {   
    
    private String strKlasse;
    
    public Schueler() {
        // Standard-Konstruktor
    }
     
    public void erfasseDaten() {
        Scanner sc = new Scanner(System.in);
        // Ausführung der Basislogik (Namen und Adresse erfassen)
        super.erfasseDaten(); 
        
        // Spezifische Erweiterung für den Schüler
        System.out.print("Klasse:\t\t");
        strKlasse = sc.next();
        System.out.println();
    }
     
    // Überschreiben (Method Overriding) der Basis-Methode
    public void druckeAusweis() {
        System.out.println("Schülerausweis");
        super.druckeAusweis(); // Druckt die allgemeinen Personendaten
        System.out.println("Klasse.:\t" + strKlasse);
        System.out.println();
    }
}

4. Subklasse: Lehrer.java

Analog zur Schüler-Klasse erbt der Lehrer von der Person, erweitert den Zustand jedoch um das Fachgebiet (strFach).

import java.util.Scanner;

public class Lehrer extends Person {
    
    private String strFach;
    
    public Lehrer() {
    }
     
    public void erfasseDaten() {
        Scanner sc = new Scanner(System.in);
        super.erfasseDaten();
        System.out.print("Fach:\t\t");
        strFach = sc.next();
        System.out.println();
    }
     
    public void druckeAusweis() {
        System.out.println("Lehrerausweis");
        super.druckeAusweis();
        System.out.println("Fach:\t\t" + strFach);
        System.out.println();
    }
}
