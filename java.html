1. Arrays und Listen

Arrays

Feste Größe, schneller Zugriff über Index.

int[] zahlen = new int[5];
zahlen[0] = 10;
int[] werte = {1, 2, 3, 4, 5};

// Array durchlaufen
for(int i = 0; i < werte.length; i++) {
    System.out.println(werte[i]);
}

ArrayList

Dynamische Größe, Teil des Collections Frameworks.

ArrayList<String> liste = new ArrayList<>();
liste.add("Java");
liste.add("Python");
liste.remove("Python");

// For-Each Schleife
for(String s : liste) {
    System.out.println(s);
}

2. Objektorientierte Programmierung (OOP)

Klassen und Objekte

Klassen sind Baupläne, Objekte sind konkrete Instanzen.

public class Auto {
    // Eigenschaften (Attribute)
    private String marke;
    private int baujahr;

    // Konstruktor
    public Auto(String marke, int baujahr) {
        this.marke = marke;
        this.baujahr = baujahr;
    }

    // Methoden (Verhalten)
    public void fahren() {
        System.out.println(marke + " fährt.");
    }
}

// Verwendung
Auto meinAuto = new Auto("VW", 2020);
meinAuto.fahren();

Vererbung

Wiederverwendung von Code durch extends.

public class ElektroAuto extends Auto {
    private int batterieKapazitaet;

    public ElektroAuto(String marke, int baujahr, int kapazitaet) {
        super(marke, baujahr); // Aufruf Konstruktor Oberklasse
        this.batterieKapazitaet = kapazitaet;
    }

    @Override
    public void fahren() {
        System.out.println("Fährt leise elektrisch.");
    }
}

3. Polymorphismus und Interfaces

Merksatz Polymorphismus (Vielgestaltigkeit) bedeutet, dass eine Variable vom Typ einer Oberklasse oder eines Interfaces auf Objekte verschiedener Unterklassen verweisen kann. 
public interface Fahrbar {
    void beschleunigen();
    void bremsen();
}

public class Fahrrad implements Fahrbar {
    public void beschleunigen() {
        System.out.println("In die Pedale treten");
    }
    public void bremsen() {
        System.out.println("Handbremse ziehen");
    }
}

// Polymorphe Verwendung
Fahrbar meinGefaehrt = new Fahrrad();
meinGefaehrt.beschleunigen();

4. Fehlerbehandlung (Exceptions)

Umgang mit Ausnahmesituationen zur Laufzeit.

Konstrukt Beschreibung
try Codeblock, der Fehler werfen könnte
catch Fängt den Fehler und behandelt ihn
finally Wird immer ausgeführt (z.B. für Ressourcen-Freigabe)
throw Wirft aktiv einen neuen Fehler 
try {
    int ergebnis = 10 / 0; // Wirft ArithmeticException
} catch (ArithmeticException e) {
    System.out.println("Fehler: Division durch Null!");
} finally {
    System.out.println("Wird auf jeden Fall ausgeführt.");
}

5. Wichtige Algorithmen

Bubblesort

Einfacher, aber langsamer Sortieralgorithmus (O(n²)).

public void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                // Tauschen
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}

Binäre Suche (Binary Search)

Schnelle Suche (O(log n)), setzt ein sortiertes Array voraus.

public int binarySearch(int[] arr, int target) {
    int left = 0;
    int right = arr.length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (arr[mid] == target) {
            return mid; // Gefunden
        }
        if (arr[mid] < target) {
            left = mid + 1; // Rechte Hälfte
        } else {
            right = mid - 1; // Linke Hälfte
        }
    }
    return -1; // Nicht gefunden
}
